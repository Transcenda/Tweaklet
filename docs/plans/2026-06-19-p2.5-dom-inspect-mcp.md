# P2.5 — Agent Self-Verification via a Widget-Backed DOM-Inspect MCP

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Give the `assistant` agent a `dom_query(selector)` tool that reads the **user's live rendered page** (via the widget) so it can verify a visual change actually appeared — and a must-verify rule so it stops claiming unconfirmed success.

**Architecture:** Tweaklet hosts a **remote MCP server** at `/tweaklet/mcp` (in its own process, using `@modelcontextprotocol/sdk`), registered in the auto-written `~/.config/opencode/opencode.json` (`mcp: { "tweaklet-dom": { type: "remote", url: "http://127.0.0.1:4319/tweaklet/mcp", enabled: true } }` — shape confirmed against the opencode schema: `McpRemoteConfig`). The tool's backend is the **widget reading the live DOM**, via the same round-trip already built for permission prompts: the tool handler emits a `dom_inspect` event on the active prompt's SSE stream + registers a pending; the widget runs `document.querySelector` and POSTs the result to `/agent/dom-result`, resolving the pending. Works because P1's booking model = one active holder → one widget → one live page.

**Tech Stack:** `@modelcontextprotocol/sdk` (MCP server, Streamable-HTTP transport mounted in the existing Express app), the existing `pendingAsks`/SSE round-trip pattern (`server.ts:141,546,583`), opencode remote MCP, the Shadow-DOM widget's host-`document` access (already used by the element picker).

**Branch:** `feature/tweaklet-dom-inspect-mcp` → PR to `main` (spans tweaklet server + web; not tweaklet-trunk). Full gate (`make ci-local`) on push.

**Builds on (now in main):** P2 live preview (#86) + failover fix (#87), per-user OAuth + agent-cwd + provider-auto-config + smoke-test (#85).

- `.opencode/agent/assistant.md` (t8a root) — the must-verify rule.
- Tests alongside each.

### Task 2: DOM-inspect round-trip primitive

**Files:** Create `tweaklet/src/agent/dom-inspect.ts` + test.

**Context:** mirrors `pendingAsks` (server.ts:141) + `onAsk` (546) + `/agent/permission` (583). A module-level active-prompt context holds the SSE `send` + a `domPending` map; `requestDomInspect` emits a `dom_inspect` event + awaits resolution.

- [ ] **Step 1 (test):** with a fake `send` + a fake resolver, `requestDomInspect("h1")` emits an event `{ type: "dom_inspect", requestId, selector: "h1" }` via `send`, and resolves with the payload passed to `resolveDomInspect(requestId, payload)`. A second test: `requestDomInspect` resolves to `{exists:false}` on timeout (configurable `timeoutMs`) if never resolved.
- [ ] **Step 2:** implement:
```ts
export interface DomResult { exists: boolean; outerHTML?: string; text?: string; computedStyle?: Record<string,string>; }
type Ctx = { send: (e: any) => void; pending: Map<string, (r: DomResult) => void> };
let active: Ctx | null = null;
export function setActivePrompt(ctx: Ctx | null) { active = ctx; }
let seq = 0;
export async function requestDomInspect(selector: string, opts: { timeoutMs?: number } = {}): Promise<DomResult> {
  if (!active) return { exists: false }; // no live widget channel
  const requestId = `dom_${++seq}`;
  return await new Promise<DomResult>((resolve) => {
    const t = setTimeout(() => { active?.pending.delete(requestId); resolve({ exists: false }); }, opts.timeoutMs ?? 15000);
    active!.pending.set(requestId, (r) => { clearTimeout(t); resolve(r); });
    active!.send({ type: "dom_inspect", requestId, selector });
  });
}
export function resolveDomInspect(requestId: string, r: DomResult): boolean {
  const fn = active?.pending.get(requestId); if (!fn) return false;
  active!.pending.delete(requestId); fn(r); return true;
}
```
- [ ] **Step 3:** run tests → pass. Commit.
