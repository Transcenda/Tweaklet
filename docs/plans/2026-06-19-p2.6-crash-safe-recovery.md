# P2.6 — Crash-Safe History + Recovery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Survive the agent breaking the very app its UI is embedded in: the conversation re-hydrates after a reload/crash (opencode is the durable store), and the user always has a working channel to tell the agent "revert/fix."

**Architecture:** opencode already persists each session's messages (verified: `GET /session/{id}/message` survives restarts). The two gaps: (1) Tweaklet's `login→sessionId` map is **in-memory** (lost on a Tweaklet restart) → make it durable; (2) the panel's activity log is built live from the SSE stream and is **lost on reload** → add a re-hydration endpoint that maps opencode's stored messages back into `AgentEvent`s and replays them on mount. The crash-proof channel (standalone `/tweaklet/`) and always-available revert (`/agent/undo`, `/agent/reject`) already exist; we add a recovery affordance + make re-hydration work in both embedded and standalone modes.

**Tech Stack:** Tweaklet (Node/TS), the existing `AgentEvent` model (`src/agent/events.ts`), opencode session messages via the SDK client, the React panel.

**Branch:** `feature/tweaklet-crash-safe-recovery` → PR to `main`. Full gate on push.

**Builds on (in main):** P1 (#85), P2 (#86/#87), P2.5 (#88).

### Task 1: Durable session store
**Files:** Create `tweaklet/src/server/session-store.ts` + test. Modify `server.ts` to use it.

- [ ] **Step 1 (test `session-store.test.ts`):** with an injected read/write (in-memory fake), `makeSessionStore` loads an existing `{alice:"ses_1"}`, `get("alice")==="ses_1"`; `set("bob","ses_2")` then `get` returns it AND the write fn was called with both entries; `delete("alice")` removes + persists; a fresh store over the same backing data survives a "restart" (re-construct → `get("bob")` still works).
- [ ] **Step 2:** implement:
```ts
import { readFileSync, writeFileSync, existsSync, mkdirSync } from "node:fs";
import { dirname } from "node:path";
export interface SessionStore { get(login: string): string | undefined; set(login: string, id: string): void; delete(login: string): void; }
export function makeSessionStore(
  path: string,
  io: { read?: () => string | null; write?: (s: string) => void } = {},
): SessionStore {
  const read = io.read ?? (() => (existsSync(path) ? readFileSync(path, "utf8") : null));
  const write = io.write ?? ((s) => { mkdirSync(dirname(path), { recursive: true }); writeFileSync(path, s, { mode: 0o600 }); });
  let map: Record<string, string> = {};
  try { const raw = read(); if (raw) map = JSON.parse(raw); } catch { map = {}; }
  const persist = () => write(JSON.stringify(map));
  return {
    get: (l) => map[l],
    set: (l, id) => { map[l] = id; persist(); },
    delete: (l) => { delete map[l]; persist(); },
  };
}
```

Keep all existing tests green (inject an in-memory store in tests if needed).
